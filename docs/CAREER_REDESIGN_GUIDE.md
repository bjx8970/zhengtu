# 《政途人生》职业与治理玩法大型改版指导

> 状态：长期产品方向指导（非当前已实现功能）
> 适用范围：bjx8970/zhengtu
> 最后校准：2026-07-21
> 改版代号：Career Framework Rebuild

## 一、改版背景

当前项目已实现角色建档、部门行动、时间推进、年度考核、晋升流程和本地存档，具备可运行的核心原型。

当前版本围绕以下逻辑展开：选择职业线 → 安排行动 → 提升 KPI 和属性 → 玩家主动发起晋升 → 通过固定晋升流程 → 提升等级。

随着职业层级升高，现有模型会产生以下问题：

- 将普通科员、领导职务、公务员职级和机构层级混合在一个 `currentLevel` 中
- 将行政、党务、纪检和群团设计为永久封闭的职业树
- 所有层级都使用相似的部门行动和晋升流程
- 玩家能够随时主动申请高级领导岗位
- 政治资本成为几乎所有职业问题的万能资源
- 国家级岗位容易退化为普通数值满足后的"下一等级"
- 最高领导职务无法通过现有单职位模型准确表达

核心目标：在保留现有工程基础的前提下，重建从普通公务员到国家级正职的完整职业生涯，并将总书记、国家主席、中央军委主席兼任设置为最高等级的隐藏传奇结局。

## 二、核心设计决策

### 1. 游戏职业上限保留至国家级正职

领导职务层次：乡科级副职 → 乡科级正职 → 县处级副职 → 县处级正职 → 厅局级副职 → 厅局级正职 → 省部级副职 → 省部级正职 → 国家级副职 → 国家级正职。

### 2. 国家级正职是终局职位池

不是一个单独职位，而是包括：中共中央总书记、国务院总理、全国人大常委会委员长、全国政协主席、中华人民共和国主席、中央军事委员会主席。

### 3. 最高隐藏结局是三职兼任

玩家同时担任中共中央总书记、中华人民共和国主席和中央军事委员会主席。游戏内部应分别保存这些职务状态，不得合并为一个职位字段。

## 三、产品定位

从"升官模拟器"调整为：中国公务员职业生涯、组织运行与公共治理模拟游戏。

玩家没有达到国家级正职，也可以获得完整且正面的结局。

## 四、职业体系重构

### 删除单一等级作为职业事实来源

将 `currentLevel` 拆分为：当前具体职位、领导职务层次、公务员职级、机构层级。保留派生字段 `contentTier` 用于内容分组。

### 职业线改为岗位领域

当前行政、党群、纪检、群团四条路线改为可交流的岗位领域：地方综合治理、党委工作机关、政府综合机关、政府专业部门、纪检监察、人大机关、政协机关、群团机关、中央综合机构、国防和国家安全决策领域。

## 五、完整职业阶段

1. 录用与试用
2. 普通公务员和业务骨干
3. 基层和科级领导
4. 县处级领导
5. 厅局级领导
6. 省部级领导
7. 国家级副职
8. 国家级正职终局

不同阶段必须具有不同核心玩法，不能只是部门行动卡片数量增加。

## 六、国家级正职终局池

- 国务院总理终局：国家行政管理、宏观经济、财政民生
- 全国人大常委会委员长终局：立法规划、监督工作、法治遗产
- 全国政协主席终局：政治协商、统一战线、社会共识
- 中华人民共和国主席终局：国家代表活动、国家仪式、外事活动
- 中共中央总书记终局：必须从中央政治局常委中产生
- 中央军委主席终局：需经历党和国家两个军委任职节点

## 七、最高传奇结局

### 达成状态

同时满足：中共中央总书记、中华人民共和国主席、党的中央军委主席、国家中央军委主席。

### 达成前置条件（必要但不充分）

已任国家级副职、已进入中央政治局常委、拥有地方和中央双重履历、有全国性重大任务成果、无严重纪律问题、NPC 竞争格局允许。

### 达成后

进入包含 3-5 项重大决定的终章，完成后进入最终评价并结束游戏。不继续运行常规循环。

## 八、晋升与岗位机会重构

- 取消玩家随时点击晋升
- 领导岗位机会由世界状态产生（空缺、换届、退休、交流、机构调整）
- 职级晋升与领导职务选拔分开
- 机会结果包括：提拔、职级晋升、平级交流、下派锻炼、上挂、后备培养、继续观察等

## 九、核心资源重构

删除政治资本万能货币，拆分为：组织信任、专业声誉、群众认可、班子支持、全国影响、廉洁与责任记录。

## 十、NPC 干部系统

高级职务必须通过 NPC 竞争形成稀缺性。NPC 可以获得晋升、交流、进入中央、在换届中进入领导班子、因考核或风险退出、与玩家竞争国家级岗位。

## 十一、五年政治周期

高级职业阶段使用五年周期组织内容。基层阶段仍可使用季度和年度循环。

## 十二、结局体系

- 普通职业结局：基层实干者、专业型公务员、高职级业务骨干等
- 高级成功结局：厅局级/省部级退休
- 国家级副职结局
- 国家级正职结局（按具体职务分别结算）
- 最高传奇结局：三职兼任

## 十三、当前代码的迁移指导

### 可以保留的部分

SolidJS + Vite、TypeScript 严格模式、Engine 纯函数原则、JSON 配置和配置验证、行动持续时间和冷却机制、行动槽位、五维考核基础、Vitest 测试、GitHub Actions 质量门禁、localStorage 原型存档。

### 必须替换的核心字段

- `currentLevel` → 当前职位 + 领导职务层次 + 公务员职级 + 机构层级 + 派生内容等级
- `currentCareerLine` → 当前岗位领域 + 历史岗位领域 + 专业方向
- `politicalCapital` → 组织信任 + 专业声誉 + 群众认可 + 班子支持 + 全国影响
- `promotionStage` → 当前职业机会 + 机会类型 + 候选职位 + NPC 候选池

### 最高职务状态

增加独立状态：是否为中央委员、政治局委员、政治局常委、总书记、党的中央军委主席、国家主席、国家中央军委主席。

## 十四、推荐实施阶段

- Phase 1：建立存档和时间基础（已完成）
- Phase 2：重构职业数据模型
- Phase 3：重做基层纵向切片
- Phase 4：NPC 和岗位机会
- Phase 5：县市和省部级内容
- Phase 6：国家级副职原型
- Phase 7：国家级正职终局

## 十五、明确不做的内容

- 最高领导职务取得后的无限国家经营
- 完整军事单位和军队职业生涯模拟
- 真实政治人物和真实地区复刻
- 通过单一随机数直接决定最高职位
- 将最高领导职务设计为普通"第12级"

## 十六、最终设计原则

- 真实结构，适度抽象
- 最高可达，但绝非必达
- 过程可经营，结果不完全可控
- 落选也是职业内容
- 高层不是低层数值放大
- 最高职务是终局，不是新手村

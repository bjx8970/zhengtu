# 政途人生 v3.0

一款以基层治理与干部成长为主题的仕途模拟游戏。

当前版本是**本地单机优先的可玩原型**，聚焦行政线 L1-L3（科员、副科、正科）的纵向体验。玩家可以完成五步建档，在 Dashboard 中安排部门行动、推进时间、完成 KPI 与年度考核，并经历一次晋升流程。连续晋升至 L3 仍是当前阶段目标。

4 条职业线、11 个级别和更完整的职位体系属于长期设计目标，不代表当前已经实现的内容。

## 当前范围

| 能力     | 当前状态                                                |
| -------- | ------------------------------------------------------- |
| 职业发展 | 行政线 L1-L3，共 10 个职位配置；连续晋升流程尚待打通    |
| 核心循环 | 行动排期 → 时间推进 → 到期结算 → KPI/考核 → 晋升        |
| 行动槽位 | 配置为主要 3 个、次要 2 个、备用 1 个；备用槽当前不可达 |
| 页面     | 启动画面、占位登录、五步建档、综合 Dashboard            |
| 存档     | 每次操作自动写入浏览器 localStorage                     |

行政职位配置在 L3 封顶，尚无 L4 晋升目标。当前 UI 在首次晋升完成后也缺少继续下一次晋升的入口，这两项会按开发路线图分阶段处理。

## 快速开始

```bash
pnpm install
pnpm dev          # http://localhost:3000
```

无需配置后端即可使用本地自动存档。

项目包含 Supabase 远程存档的基础读写代码，但认证、启动时远程加载和本地/远程存档仲裁尚未接入完整产品流程。以下环境变量目前仅供相关功能开发使用：

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

## 质量检查

```bash
pnpm run ci        # 一键全检：format + lint + typecheck + test + validate:config + build
pnpm dev           # 开发服务器（HMR）
pnpm build         # 生产构建
```

## 长期目标

在当前纵向切片稳定后，项目计划逐步扩展：

- 将行政线从 L3 扩展至 L11（正部级）。
- 增加党务、纪检和群团职业线。
- 完善关系、上级互动、风险、个人生活和历史评价等系统。
- 在认证和存档恢复流程稳定后提供用户级云存档。

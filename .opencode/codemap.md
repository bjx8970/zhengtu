## Quick Reference

| Item                | Value                                                                      |
| ------------------- | -------------------------------------------------------------------------- |
| Language            | TypeScript 5.x (strict mode)                                               |
| Framework           | SolidJS 1.9 (NOT React)                                                    |
| JSX                 | `vite-plugin-solid` (no `@jsxImportSource` pragmas)                        |
| State               | `createStore` + `dispatch(action)` in `src/store/game-store.ts`            |
| Router              | `@solidjs/router` v0.15                                                    |
| Build               | Vite 6, output to `dist/`, base `/zhengtu/`                                |
| Package manager     | pnpm                                                                       |
| Test runner         | vitest 3 + jsdom                                                           |
| Coverage thresholds | engine >=90%, config >=80%, store >=70%                                    |
| Lint                | ESLint v9 flat config (`eslint.config.js`) + `eslint-plugin-solid`         |
| Format              | Prettier 3                                                                 |
| Config validation   | zod schema via `scripts/validate-config.ts`                                |
| CI check            | `pnpm run ci` (format → lint → typecheck → test → validate:config → build) |
| Deployment          | GitHub Pages (`deploy.yml` on push to main)                                |
| Backend             | Supabase (save persistence, `upsertSave`)                                  |
| Branch workflow     | feature branches → PR → squash merge to main                               |
| Architecture        | **UI → Store → Engine → Config** (top-down, engine has no DOM/globals)     |
| File naming         | `kebab-case.ts`, tests: `*.test.ts` (no `.spec.ts`)                        |
| Config editing      | JSON templates only; no code changes needed for value tweaks               |

## Key Patterns

- **Store mutations**: all state changes go through `dispatch(action)` → `reduceGameState(draft, action)`. Add new action types to `GameAction` union first.
- **Engine purity**: no DOM, no global state, no store references. Engine files <= 200 lines.
- **Config singletons**: `getConfigLoader()` returns singleton; templates load from JSON, references expanded at runtime.
- **Slot-based actions**: primary=3, secondary=2, reserve=1 slots. No AP system. Category rules (major/minor/routine) determine slot eligibility and cooldowns.
- **Persistence**: localStorage on every action; Supabase sync on `ADVANCE_TIME`. Load arbitration via `selectNewer(localSave, remoteSave)`.
- **List rendering**: `<For each={list}>` instead of `Array.map()` (Solid convention, enforced by ESLint).
- **ESLint config**: `eslint.config.js` flat config v9 — NOT `.eslintrc.json`.
